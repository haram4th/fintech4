    • 0️⃣ Ubuntu란 무엇인가?

    Ubuntu(우분투)는 전 세계에서 가장 널리 사용되는 리눅스(Linux) 기반 운영체제 중 하나이다.

    개발자, 서버 관리자, 기업, 연구소, 클라우드 환경에서 표준처럼 사용된다.

    ✔ Ubuntu의 주요 특징

    🔹 1) 무료 & 오픈소스

  • 누구나 자유롭게 다운로드, 설치, 사용, 수정 가능
  • 전 세계 개발자 커뮤니티가 함께 유지·보수

    • 🔹 2) 안정성과 보안성이 높다

  • 서버용 OS로 많이 사용되며, 이미 검증된 안정성
  • 계정/권한 구조가 명확해서 보안 설정이 쉽다
  • 바이러스·랜섬웨어에 상대적으로 덜 취약

    • 🔹 3) 터미널(명령어) 기반 작업에 최적화

  • GUI(그래픽)도 있지만, 개발·서버 관리에서는 터미널이 핵심
  • 자동화, 스크립트, 원격 관리에 매우 강력

    • 🔹 4) 서버·클라우드에서 사실상 표준

  • AWS, GCP, Azure 기본 이미지에 Ubuntu가 포함
  • 웹 서버, DB 서버, AI 서버 등 대부분 리눅스 기반

    • 🔹 5) 개발 생태계가 풍부

  • Python, Node.js, Java, Go, Rust 설치가 쉬움
  • Docker, Kubernetes, Git, Nginx 등과 궁합이 좋음

    • 🔟 권한과 소유권 — ls -l, chmod, chown

    리눅스에서는 소유자 / 그룹 / 기타 사용자에 대해

    읽기(r) / 쓰기(w) / 실행(x) 권한을 구분해서 관리한다.

    1️⃣ 권한 표기 형식 이해하기

    ls -l

    예시:

    -rwxr-xr-- 1 user dev  1024 Nov 17 10:00 run.sh
drwxr-xr-x 2 user dev  4096 Nov 17 09:00 mydir

    앞의 10글자를 나누면:

    - rwx r-x r--
|  |   |   |
|  |   |   └─ other(기타 사용자) 권한
|  |   └──── group(그룹) 권한
|  └──────── owner(소유자) 권한
└─────────── 파일 타입(-: 파일, d: 디렉터리)

    2️⃣ r, w, x → 숫자 권한

  • r = 4 (read)
  • w = 2 (write)
  • x = 1 (execute)

    • 예:

  • rwx = 4+2+1 = 7
  • rw- = 4+2 = 6
  • r-x = 4+1 = 5
  • r-- = 4

    • 그래서:

  • chmod 755 파일 → rwx r-x r-x
  • chmod 644 파일 → rw- r-- r--

    • 3️⃣ chmod로 권한 변경

    🔸 숫자 방식

    chmod 755 run.sh    # 소유자: rwx, 그룹: r-x, 기타: r-x
chmod 644 memo.txt  # 소유자: rw-, 그룹: r--, 기타: r--
chmod 600 secret.key # 소유자만 읽기·쓰기

    🔸 기호 방식

    chmod u+x script.sh    # 소유자(u)에 실행(x) 권한 추가(+)
chmod g-w script.sh    # 그룹(g)에서 쓰기(w) 제거(-)
chmod o-r secret.txt   # 기타(o)에서 읽기(r) 제거(-)
chmod a+r sample.txt   # 모두(a)에 읽기(r) 추가(+)

    1️⃣1️⃣ 서비스 관리 — systemctl + SSH 서버

    우분투에서는 여러 프로그램이 서비스(daemon) 형태로 백그라운드에서 동작한다.

    이 서비스들은 systemd라는 시스템에 의해 관리되고, 우리는 systemctl 명령으로 제어한다.

    1️⃣ SSH 서버란?

  • SSH(Secure Shell) : 원격으로 다른 컴퓨터에 안전하게 접속하는 프로토콜
  • 서버 쪽에서 SSH 서버(데몬) 가 실행 중이어야 접속 가능
  • Ubuntu에서는 보통 openssh-server 패키지를 설치

    • 2️⃣ SSH 서버 설치

    sudo apt update
sudo apt install openssh-server

    설치 후, 자동으로 ssh 서비스가 실행된다.

    3️⃣ SSH 서비스 상태 확인

    systemctl status ssh
  • Active: active (running) 이면 정상 동작
  • inactive / failed 상태면 접속이 되지 않는다.

    • 4️⃣ SSH 서비스 시작 / 중지 / 재시작

    sudo systemctl start ssh      # SSH 서비스 시작
sudo systemctl stop ssh       # SSH 서비스 중지
sudo systemctl restart ssh    # SSH 서비스 재시작

    sshd로 표기되는 환경도 있으나, Ubuntu 기본 서비스 이름은 ssh이다.

    5️⃣ 부팅 시 자동 실행 설정

    sudo systemctl enable ssh     # 부팅 시 자동 시작
sudo systemctl disable ssh    # 자동 시작 해제

    6️⃣ SSH 로그 확인 — journalctl

    sudo journalctl -u ssh        # ssh 서비스 관련 로그 확인
sudo journalctl -u ssh -f     # 실시간 로그 모니터링 (tail -f 느낌)
  • SSH 접속/인증 오류, 서비스 에러 등을 확인할 때 유용

    • > ⚠ WSL 주의사항

    > WSL2(Ubuntu on Windows)에서는 SSH 서버를 열어 외부에서 접속하려면

    > Windows 방화벽, 포트포워딩 등의 추가 설정이 필요하다.

    > 단순한 학습 용도라면 “WSL 내부에서만” 테스트하는 것으로도 충분하다.

    1️⃣2️⃣ 방화벽(UFW) — SSH와 포트 제어

    방화벽(Firewall) 은 외부에서 들어오는/나가는 네트워크 트래픽을 제어하는 시스템이다.

    Ubuntu에서는 기본 방화벽 도구로 UFW(Uncomplicated Firewall) 를 제공한다.

    > 클라우드(AWS, GCP 등)에서는 보안 그룹(Security Group)과 함께 고려해야 한다.

    1️⃣ UFW 상태 확인

    sudo ufw status
  • inactive : 아직 사용하지 않는 상태
  • active : 방화벽이 동작 중

    • 2️⃣ UFW 활성화/비활성화

    sudo ufw enable     # 방화벽 켜기
sudo ufw disable    # 방화벽 끄기

    > ⚠ SSH로 원격 접속 중인 서버에서 ufw enable 할 때는

    > 반드시 22번 포트를 미리 허용한 뒤 실행해야 접속이 끊기지 않는다.

    3️⃣ 포트 허용 (allow)

    🔹 숫자 포트로 허용

    sudo ufw allow 22           # SSH
sudo ufw allow 80           # HTTP
sudo ufw allow 443          # HTTPS
sudo ufw allow 8000/tcp     # 8000번 TCP 포트

    🔹 서비스 이름으로 허용

    sudo ufw allow ssh
sudo ufw allow 'Nginx Full'

    4️⃣ 포트 차단 (deny)

    sudo ufw deny 80

    5️⃣ 룰 목록 번호와 함께 확인 / 삭제

    sudo ufw status numbered     # 번호가 붙은 룰 목록 보기
sudo ufw delete 3            # 3번 룰 삭제

    6️⃣ 간단한 SSH 보안 예시

    1. SSH 허용:

        sudo ufw allow 22/tcp

    2. UFW 활성화:

        sudo ufw enable

    3. 상태 확인:

        sudo ufw status verbose

    4. 특정 IP만 SSH 허용 (예시):

        sudo ufw deny 22
    sudo ufw allow from 1.2.3.4 to any port 22 proto tcp
